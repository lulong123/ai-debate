## Vision

### 10x Check
把 AI 辩论从"聊天室 + 文字流"变成"像素风游戏场景"——用户看到的是圆桌旁的像素小人们在激烈辩论, 有气泡对话、情感反应、环境氛围, 而不是一串文字消息。这是从"工具"到"体验"的跨越。

### Platonic Ideal
用户打开页面看到一个温暖的像素风酒馆。木桌旁坐着像素小人。主持人手持法槌开场, 辩手轮流站起发言, 数据研究员掏出放大镜搜索资料。蜡烛摇曳, 灰尘飘动。辩论结束, 主持人敲法槌, 卷轴展开显示裁决。

## Core Decisions

| # | Decision | Choice | Rationale |
|---|----------|--------|-----------|
| 1 | Rendering Tech | PixiJS (WebGL-backed 2D) | 2D API + GPU acceleration + native sprite/animation support |
| 2 | Art Style | Pixel Art | AI-gen maturity highest, smallest asset size, fits 2D paradigm |
| 3 | React Integration | Imperative PixiJS + useRef | @pixi/react beta 风险过高; 直接用 PixiJS API + useRef 桥接 React 生命周期, 更可控 |
| 4 | Audio Library | Howler.js | 成熟的 Web Audio API 封装, PixiJS 无内置音频 |
| 5 | Perspective | Layered parallax (not true isometric) | "2.5D" 通过层级视差实现(前景/中景/背景分层), 非真正的等轴测投影; 实现更简单, 视觉效果足够 |

## State Machine: Character State Transitions

```
                              agent_message_start
                     ┌─────────────────────────────────┐
                     │                                 ▼
    ┌──────┐  agent_thinking  ┌──────────┐     ┌──────────┐
    │ IDLE │ ──────────────▶ │ THINKING │     │ SPEAKING │
    └──┬───┘                 └────┬─────┘     └────┬─────┘
       │                          │                 │
       │  score_update            │ timeout         │ agent_message_complete
       │ (other agent)            │ (30s)           │
       ▼                          ▼                 ▼
  ┌──────────┐              ┌──────┐          ┌──────────┐
  │ REACTING │              │ IDLE │          │ COOLDOWN │
  └────┬─────┘              └──────┘          └────┬─────┘
       │ timeout (3s)                              │ timeout (1s)
       ▼                                           ▼
  ┌──────┐                                    ┌──────┐
  │ IDLE │                                    │ IDLE │
  └──────┘                                    └──────┘

Interrupt: round_start / discussion_end → ALL characters → IDLE
```

State definitions:
- **IDLE**: Default seated position, subtle breathing animation
- **THINKING**: Thought bubble with animated dots, character looks up
- **SPEAKING**: Character stands, speech bubble expands, spotlight on
- **REACTING**: Emotional response to other agents (shock/agreement/disagreement)
- **COOLDOWN**: Brief post-speak sit-down animation before returning to idle

## SSEGameBridge Spec

```
Interface SSEGameBridge {
  // Called by React on each SSE event
  onEvent(event: SSEEvent): void

  // Returns current scene state for UI overlay
  getState(): { currentSpeaker, currentRound, status, scores }

  // Event queue: ring buffer (max 200 events)
  // High-frequency throttling: agent_message_chunk coalesced
  //   (process latest chunk only per animation frame)
  // Low-frequency events: processed immediately

  // Reconnection: on SSE reconnect, React re-feeds all buffered
  // events from useSSE. Bridge rebuilds scene state from full
  // event array (idempotent state reconstruction).
}
```

## Mobile Performance Detection

- **Detection**: Monitor PixiJS Ticker FPS. If average FPS < 24 for 30 consecutive frames → auto-disable particles and cap at 30 FPS
- **Override**: Settings toggle in GameUI overlay (user can manually re-enable)
- **Detection timing**: Check after first 5 seconds of game scene load

## Asset Pipeline

- **Sprite format**: Aseprite JSON hash (texture atlas with frame metadata)
- **Loading sequence**: Preload core sprites (moderator, table, background) → lazy-load debater sprites on `discussion_start`
- **Bundle size target**: < 2 MB total for all sprite assets (pixel art is naturally small)
- **Character reuse**: Base sprite template with color palette swap for debaters (reduces unique frames from 270 to ~90)
- **Tools**: AI generate single frames → Aseprite assemble sprite sheets → TexturePacker export

## Audio Architecture

- **Library**: Howler.js (not @pixi/sound — PixiJS audio has React compatibility issues)
- **Autoplay policy**: First user interaction (click "start debate" button) → unlock Web Audio context → all subsequent sounds play normally
- **Sound files**: 8-bit WAV/MP3, < 50 KB each, loaded on demand
- **Mute toggle**: Always visible in GameUI overlay

## Migration Strategy

- **Approach**: Feature flag (`VITE_GAME_VIEW=true` env variable)
- **Default**: Game view ON for new users, text view available via toggle
- **Toggle**: Button in header switches between `GameCanvas` and legacy `ChatStream`
- **Legacy preservation**: Current `ChatStream.tsx`, `ScorePanel.tsx`, `DataPoolPanel.tsx` kept as-is, not deleted
- **Rollout**: Feature flag → full replacement in next version after validation

**Total unique frames: ~95** (vs 270 in naive approach). Palette swap for debaters: same base sprite, different color overlay via PixiJS tint.

## Key Risks

1. **Art assets** - Even with palette swap, ~95 frames of pixel art need AI generation + manual assembly. Mitigate: AI generate key poses, interpolate states programmatically where possible.
2. **Mobile performance** - WebGL on low-end phones. Mitigate: FPS monitoring + auto-disable particles + 30 FPS cap.
3. **Pixel font rendering** - PixiJS Canvas text needs bitmap font solution. Mitigate: use `@pixi/text-bitmap` with a pixel font like "Press Start 2P".
4. **Web Audio autoplay** - Browsers block auto-play. Mitigate: unlock on first user interaction (debate start button), Howler.js handles the rest.

## Backend Impact

If this data is not already in the event, a small backend addition (3 lines) is needed. The existing `orchestrator.py` already has this data at discussion start time.
